好啦，開始說明吧

Image Defense Tool 是一套 **用於圖片展示與防護的工具**，協助創作者在公開展示圖片時，降低圖片被未經授權蒐集、挪用或作為 AI 訓練資料的風險，同時快速產生可部署的靜態圖片展示頁面。

本工具提供 **GUI（EXE）操作介面**，不需程式背景即可使用。

---

### 功用與目的

#### 主要目的
- 防止圖片被直接用於 AI 模型訓練
- 保護創作者的原始作品
- 提供圖片展示方式
- 快速建立可部署的圖片展示頁

#### 主要功能
- 自動生成展示用圖片
- 自動生成可裁切縮圖
- 產生單頁靜態圖片展示網站
- 可直接部署於 GitHub Pages / Cloudflare Pages

---

### 操作步驟（GUI 版本）

#### 1️⃣ 啟動程式
執行：
Image Defense Tool.exe

---

#### 2️⃣ 選擇圖片資料夾
- 點擊 **Browse**
- 選擇包含圖片的資料夾（JPG / PNG）

---

#### 4️⃣ 縮圖設定（Thumbnail）
- **Aspect Ratio**
  - `original`：維持原比例
  - `1:1` / `16:9` / `4:3`：自動裁切
- **Max Size**
  - 設定縮圖最大邊長（像素）

---

#### 5️⃣ 網站設定
- **Site Title**：展示頁標題
- **Base URL**：部署網址（用於 Open Graph / 分享）

---

#### 6️⃣ 聲明設定（Notice）
- 可選擇是否顯示圖片使用聲明
- 可自訂：
  - Title
  - Body
  - Extra

---

#### 7️⃣ 建立展示頁
- 點擊 **Build Website**
- 程式完成後，會輸出一個靜態網站資料夾

---

### 輸出結構

最終產生的展示頁資料夾結構如下：  
IMG/  
├─ index.html # 圖片展示主頁  
├─ image.bat # 圖片本圖
├─ og_preview.jpg # 社群分享預覽圖  
├─ viewer.js # 圖片顯示與互動邏輯  
└─ style.css # 網站樣式  


📌**說明**
- `index.html`：主展示頁
- `og_preview.jpg`：分享連結時的預覽圖
- `viewer.js`：控制圖片瀏覽、互動行為
- `style.css`：版面與顯示樣式

---

### 部署方式（簡述）

#### GitHub Pages
1. 建立 GitHub Repository
2. 將 `IMG/` 內的檔案上傳
3. 啟用 Pages（Root 或 `/docs`）

#### Cloudflare Pages
1. 建立 Pages 專案
2. 上傳 `IMG/` 作為輸出目錄
3. 不需額外建置指令
